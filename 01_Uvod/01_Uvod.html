Analiza podatkov

  • Potrebe po podatkovnih analitikih (podatkovni inenir, ang. data scientist) strmo naraajo

Kdo rabi podatkovne analitike?

  • Finanni sektor (banke, zavarovalnice, fintech, …)
  • Nova hitro rastoa start-up podjetja temeljea na informacijskih reitvah:
    • deljena ekonomija: Uber, Airbnb, …
    • spletni marketing: Google, Facebook, …
    • internet stvari, velika podatkovja, mobilne aplikacije, verienje blokov, …
  • Podatkovno-krmiljena podjetja (ang. data driven), digitalna transformacija, …
  • AI - umetna inteligenca, strojno uenje

Kaj delajo podatkovni analitiki?

  • Postavljajo cilje, izzive, hipoteze
  • Definirajo vrste potrebnih podatkov
  • Ugotavljajo, kakne podatke lahko dobijo
  • Pridobivajo podatke
  • istijo podatke
  • Izvajajo raziskovalne analizo (ang. exploratory analysis)
  • Sestavljajo predikcijske modele
  • Interpretirajo rezultate
  • Kritino vrednotijo rezultate
  • Iz rezultatov potegnejo zakljuke in jih ustrezno zapiejo
  • Poskrbijo za ponovljivost analiz (dokumentacija, koda, verzije, …)
  • Objavijo rezultate in jih distribuirajo

Osnovni podatkovni tipi

  • Vsak podatkovni objekt, ki ga lahko priredimo v spremenljivko ima tip (razred)

  • character - nizi ("To je besedilo", 'to je niz')
  • numeric - realna tevila (1.2, -3.55, 1.1e-3)
  • integer - cela tevila (1, -20)
  • complex - kompleksna tevila (2 + 3i)

  • logical - logine vrednosti (TRUE, FALSE)

Sestavljeni podatkovni tipi

  • vector - vektorji so tabele (tabelarini seznami) objektov istega osnovnega tipa
  • Minimalni uporabni objekti so vektorji
  • Tudi e vnesemo tevilo 1, je to vektor tipa numeric doline 1
  • Izpeljanke tipa vector
    • matrix - matrike, vektorji s podanim razbitjem po vrsticah
    • array - vedimenzionalne matrike/tabele
    • factor - vektorji kategorinih spremenljivk (omejen nabor vrednosti)
  • list - seznami lahko vsebujejo objekte razlinih podatkovnih tipov
  • data.frame - razpredelnice - dvo-dimenzionalne podatkovne tabele s poimenovanimi stolpci kot vektorji (standardna oblika podatkov za izvajanje analiz)

Prirejanje in spremenljivke

  • imena spremenljivk: zaporedja rk, tevk, pik in podrtajev

  • operatorji prirejanja (<-, ->, =)
  • prirejanje je imenovanje objektov v danem okolju (environment)
  • objects() in ls() za izpis objektov po imenih v okolju (izpis spremenljivk)
  • rm(...) brisanje poimenovanj in objektov (spremenljivk) v okolju

    Sestavljeni izrazi

    • Sestavljen izraz je zaporedje izrazov loenih s podpiji v zavitih oklepajih, ki se izvedejo
    • Rezultat zaporedja je vrednost zadnjega izraza (e ni za njim podpija)
    • > {a <- 1; b <- 2 + a; a + b}
[1] 4
    • Sestavljene izraze uporabljamo podobno kot pri Pythonu zamikanje (bloki), tipino pri krmilnih stavkih, zankah in funkcijah

    Pogojni stavek (if)

    • > a <- 1
> if (a > 0) print("veji") 
[1] "veji"
> if (a == 5) print("Je pet.") else print("Ni pet") 
[1] "Ni pet"
    • Pogoj je vedno v oklepaju za if, else pa ni obvezen
    • Namesto zamikanja uporabljamo bloke
    • > a <- 1
> if (a < 3) {a <- a + 1; a <- a + 1; print("Morda pa je zdaj ez tri ...")} 
[1] "Morda pa je zdaj ez tri ..."
    • Pri R zamikanje ni pomembno - imamo sestavljene izraze (oz. bloke), ki jih lahko lepo razpiemo ez ve vrstic.
    • Vseeno zamikamo, da je koda bolj pregledna. Npr. v urejevalniku zapiemo:
    • if (a < 3) {
    a <- a + 1; 
    a <- a + 1;   
    print("Morda pa je zdaj ez tri ...")
}

    Zanki while in repeat

    • Zanka while se obnaa enako kot v Pythonu
    • > i <- 0
> while (i < 3) {print(i); i <- i + 1}
[1] 0
[1] 1
[1] 2
    • Zanka repeat je neskonna zanka. Prekinemo jo lahko z break
    • > i <- 0
> repeat {print(i); i <- i + 1; if (i > 2) break}
[1] 0
[1] 1
[1] 2

> gcd(12, 15)
[1] 3

    Funkcije

    • Funkcije so tudi objekti (shranimo jih v spremenljivke in jim tako dodelimo ime)
    • Zato jih lahko podajamo kot parametre, vraamo, gnezdimo, …
    • Funkcije imajo lahko pozicijske argumente in poimenovane parametre
    • Ujemanje argumentov in parametrov je malce drugae kot pri Pythonu:
    • Najprej se doloijo poimenovani parametri
    • Potem se nepoimenovane argumente obravanava po vrstnem redu pojavitve
    • Argument "tri pikice" (...) pomeni zaporedje nekih argumentov. Tipino "tri pikice" predamo kot prvi ali zadnji argument v klicu neke notranje funkcije.
